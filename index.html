Architecture & Logic

This Grandmaster Chess engine is built using a purely functional approach to game state management. The application utilizes a 2D matrix (8x8 array) to represent the literal board state, where each cell identifies the piece type and color. The move validation system is strictly deterministic, ensuring that every move follows the official FIDE chess rules. This includes pawn-specific double-moves on start, knight-L shapes that ignore blocking pieces, and infinite-range pathfinding for bishops, rooks, and queens.

For the Artificial Intelligence (Vs Computer) mode, the system implements a recursive logic pattern. While current v1.0 uses a prioritized move-evaluation heuristic (favoring captures and center-control), it is architected to support the Minimax algorithm with Alpha-Beta pruning in future updates. The AI analyzes the board, identifies all valid legal moves for the black team, and selects the optimal continuation based on piece-value weights: Queen (9), Rook (5), Bishop (3), Knight (3), and Pawn (1).

The User Interface is designed with Glassmorphism principles, featuring high-blur backdrops and subtle border glows that give the match a premium, futuristic feel. The wooden board texture is a homage to the traditional roots of the game, integrated perfectly into a digital ecosystem. Every interaction, from piece selection to move logging, is handled via asynchronous event listeners to ensure zero-latency performance even on mobile browsers.

Technical Implementation

The project uses a sophisticated turn-based state machine. When a piece is selected, the engine dynamically generates a "Valid-Move Matrix" to guide the user. The coordinate system translates raw grid indices (0-7, 0-7) into standard algebraic notation (e.g., e2 to e4) for the professional log display. Responsive design is a core priority; the board area uses the CSS aspect-ratio and calc() functions to maintain a perfect square across all viewport sizes, from 4K desktops to compact smartphones.
